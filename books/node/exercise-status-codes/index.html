# Exercise status codes

Follow me on Twitter, happy to take your suggestions on topics or improvements /Chris

In this exercise, we will show how you can use different status codes and apply them in correct situations.

We will learn how to:

  • Set a status code for a found resource
  • Handle a request when a resource can't be found
  • Protect a resource, we will learn how we can work with credentials and respond appropriately when that credential is missing
  • Work with redirecting the user if a resource has temporarily moved

# Exercise - Found and missing resources

In this exercise, we will create two files server.js and client.js. server.js will contain our server application. Let's start by adding some code to server.js and then explain what's going on:

const http = require('http');
const PORT = 8000;

const server = http.createServer((req, res) => {
  if(req.url == '/') {
    res.writeHead(200, { 'Content-Type': 'text/plain' })
    res.end('Everything is OK');
  } else {
    res.writeHead(404, { 'Content-Type': 'text/plain' })
    res.end('Not found')
  }
})

server.listen(PORT, () => {
  console.log(`Server up and running on ${PORT}`)
})

# Exercise - Protect resources

In this exercise, we will learn to create a server where some resources are protected. If the user is able to come up with acceptable credentials, then they will be permitted to access the resources. If not, then the user is to be shown a 401 status code and an appropriate message.

We will do this exercise in the following steps

  • Create a server, we will add logic for protecting resources under /products with a secret. Secrets are passed through the Authorization header
  • Create a client
    • Call without Authorization header, we will first call without an appropriate and therefore be given a 401 status code back
    • Call with Authorization header, here we will provide the header with the correct value matching the secret on the server-side

# Create a server

First, let's create a new file server-protected.js. Next, give it the following content:

const http = require('http');
const PORT = 8000;

const server = http.createServer((req, res) => {
  console.log('Asked for url', req.url);

  if (req.url == '/') {
    res.writeHead(200, {
      'Content-Type': 'text/plain'
    })
    res.end('Everything is OK');
  } else if(req.url == '/products') {
      let authCode = req.headers['Authorization'];
      if (authCode === 'secret123') {
        let products = [{
          id: 1,
          name: 'compass'
        }]
        res.writeHead(200, { 'Content-Type' : 'application/json' });
        res.end(JSON.stringify(products));
      } else {
        res.writeHead(401, { 'Content-Type' : 'text/plain' });
        res.end('Please log on to access this resource');
      }
  } else {
    res.writeHead(404, {
      'Content-Type': 'text/plain'
    })
    res.end('Not found')
  }
})

server.listen(PORT, () => {
  console.log(`Server up and running on ${PORT}`)
})

# Exercise - Work with re-directions

We've learned much already but let's see how we can work with redirects.

The idea with a redirect is for the server to respond back to the client that this resource has been moved. Additionally the server should provide information on where the resource can be found instead. There are two ways we could be talking to our server:

  1. A Browser, a browser will, when given a redirect try to make an additional request towards the new URI that is indicated
  2. Code, with the http module there is no automatic following behavior. For that reason, we need to write code that redirects our initial request to the new indicated one

### Building our Server.

Let's implement the server part. Create a file server-redirect.js and give it the following content:

const http = require('http');

const server = http.createServer((req, res) => {
  console.info('URL', req.url);
  if (req.url === '/redirect') {
    res.writeHead(303, { 'Location' : 'http://localhost:8000/temp' });
    res.end('This is not the droid you are looking for');
  } else if (req.url === '/temp') {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('This IS the droid you are looking for');
  }
});

server.listen('8000', () => {
  console.log('Server running on 8000');
})

# Using code

As we stated above, with code and the http module we don't have this automatic behavior but we can still listen for a 303 code and take appropriate measures.

Create a client-redirect.js file. Give it the following content:

const http = require('http');

function innerRequest(url) {
  http.get(url, (res) => {
    console.info(`Status ${res.statusCode}`);

    res.on('data', (chunk) => {
      console.log(`Chunk: ${chunk}`);
    });

    res.on('end', () => {
      console.log('end of stream');
    })
  })
}

http.get('http://localhost:8000/redirect', (res) => {
  console.info(`Status ${res.statusCode}`);

  if (res.statusCode == '303') {
    innerRequest(res.headers.location);
  }

  res.on('data', (chunk) => {
    console.log(`Chunk: ${chunk}`);
  });

  res.on('end', () => {
    console.log('end of stream');
  })
})
