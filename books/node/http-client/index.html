# HTTP Client

Follow me on Twitter, happy to take your suggestions on topics or improvements /Chris

So far we have been using the Browser and code as Clients to talk to our server. There are more things we can do as a client than just receiving data. Other things we can do could be sending data to our server.

There are many ways we could be sending data to a backend server application

  1. Browser, we'll render a form and then we'll allow the user to input data into the form. Lastly the user will be able to submit the form to the server application
  2. Code, we'll make a POST request from code and see the server application receive it
  3. 3rd party clients. There are many popular 3rd party clients like cURL, for example that can also post data

How we usually send data through a browser is by constructing a form with input fields. A form can have different types of input elements in it, everything from text fields, radio buttons to even elements that allows you to submit a file from your file system.

Sending simple text fields versus sending a file differs a bit so let's treat this as different scenarios. We therefore have the following scenarios that we will cover:

  1. Scenario 1, posting data from different text fields. In this scenario we will declare a form with a number of input fields of varying types except for the file type.
  2. Scenario 2, posting a file. In this scenario we will use an input element that allows us to send a file.

# Scenario 1 - posting data from text fields

What we need in this scenario are two things:

  1. A form the user can interact with and fill in values in different input fields.
  2. Code on the server that is able to respond to a form being submitted.

### The form

The form we are about to create will need to have a form tag and a few input elements declared inside of the form tag. It will also need a button to be able to submit the form to the server.

Let's showcase what such a form might look like below:

<form action="/submit" method="POST">
  <input name="city" placeholder="city" />
  <input name="country" placeholder="country" />
  <input name= "address" placeholder="address" />  
  <button>Submit</button>  
</form>

Above we have declare the following:

  • form we have a form tag with two attributes set, action and method. action is pointing to the URL we want to send the submitted form to. method is the chosen HTTP verb. By assigning it the value POST we are saying that we want send data across that will create a resource on the server
  • input we have three different inputs. We have set their name attribute to a unique identifier and we have also populated the placeholder. The placeholder is a default text that will show inside of the input element to convey to the user what type of information it should be showing.
  • button, the last element in the form is a button. When a button is placed inside of a form it takes on the role of being the element that trigger the form to send it's data to where the action property points.

We are all set up and can now concentrate on the backend.

# Scenario 2 - posting a file

To support the posting of a file there are two things we need to do:

  1. Declare a form containing an input element of type file
  2. Write server code that listens to the incoming request

# The form

For the form tag there are three attributes we need to set:

  1. action, we should set this to unique route on the server so we are able to distinguish between different requests.
  2. method, we should set this to the value POST as we mean to send data.
  3. enctype, we need to set this to the value multipart/form-data when we mean to send a file across.

Let's define below what such a form could look like below:

<form enctype="multipart/form-data" action="/file" method="POST">
  <input type="file" name="file" placeholder="select a file" />
  <button>Submit</button>  
</form>

What we most likely want to do with a file being sent to the server is to persist it to the file system. What was sent across is, although readable, something we need to parse. If we want the name of the file being submitted we would need to parse that out of line 2. We most likely want the file content so we would need to dig that out from what was transmitted above.

A lot of Node.js developers tend to rely on a 3rd part library for this parsing. One of the more popular libraries that can help you with that is multer. Have a look here if you are interested https://www.npmjs.com/package/multer to know more.
