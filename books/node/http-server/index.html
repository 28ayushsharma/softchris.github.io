# HTTP Server

Follow me on Twitter, happy to take your suggestions on topics or improvements /Chris

In the module http we can use the following classes to work with the server-side:

  • Server, this represents an instance of a HTTP Server. This is what we will work with most to start listening to different events.
  • IncomingMessage, this is a readable stream created either by http.Server or http.ClientRequest. You can use it to access things such as status, headers and data
  • ServerResponse, this is a stream created internally by the HTTP Server. This class enables you to define what the response should look like in terms of what headers it should have, what the response should be and so on.

Now we know three major classes involved in creating the server-side but how do they work together? Let's learn the how by example.

# Our first HTTP Server

The best way to learn how our three mentioned classes above work is by creating an HTTP Server. Let's create that server. To do so first we need to import our http module like so:

const http = require('http');
1

Next up we need to call createServer(). Let's see the rest of the code:

const http = require('http');
const PORT = 3000;

const server = http.createServer((req, res) => {
  res.writeHead(200, {'Content-Type': 'text/plain'});
  res.end('hello world');
});

server.listen(PORT, () => {
  console.log(`listening on port ${PORT}`)
})

Looking at the code above we first have the method createServer(). When invoked createServer() create an instance of type Server. Furthermore createServer() takes a function as an argument. The function has two input arguments. The first parameter req is of type IncomingMessage. The second argument req is of type ServerResponse. We can see that the first thing we do is call res.writeHead(). This will set the response header type to text/plain, which means simple text. We will show how to use different types of response type later on in the module. The last thing we do is to call res.end(). This writes to the ServerResponse and will be something the client will be sent. The very last line in the program is server.listen(). Invoking this method makes it possible to listen to our created HTTP Server. As part of calling server.listen() we give it a port to listen to and a callback function that will be invoked as soon as the HTTP Server is up and running. Executing this code you should have something like this showing in the terminal:

listening on port 3000

# Consuming our HTTP Server

There are two ways to connect to this HTTP Server

  1. A browser, browsers are made for this. They speak the HTTP protocol
  2. Code, we can write code that programmatically is able to connect to this HTTP Server

# Using code

To connect programmatically to our HTTP Server we can use the http module. There are different methods on the module that can help us achieve that. The most important one is the request() method. This method has several overloads. One version of it takes an option object that can look like this:

const options = {
  port: 3000,
  host: 'localhost',
  headers: {},
  method: 'GET'
};

Information like portand host is so we know where to go with our request. headers are further instructions we give to the server on how we wish to connect, what type of content is coming and so on. method signals what intention we have. With intention we mean if we want to read, write, update or maybe delete resources on the server.

Let's look at some basic client code:

const http = require('http');
const PORT = 3000;

http.get(`http://localhost:${PORT}`, (res) => {
  res.on('data', (chunk) => {
    console.log(`DATA: Chunk received: ${chunk}`)
  })

  res.on('end', () => console.log('END: no more data'));
  res.on('close', () => {
    console.log('CLOSE: Connection closed')
  })

  res.on('error', (err) => console.error(err.stack));
});

Above we are calling get(). This is a convenience method that underneath calls request(). The difference between get() and request() is that the header property method have been set to GET, signaling we want to read data. Further inspection of our get() method we see that we pass it a URL and a callback function. The callback function takes a parameter res, which is a response stream. There are several events on this response stream we can and should listen to:

  • data, this event is raised every time the server has sent a new chunk of data. Every chunk is by default 16 bytes, so depending on the size of what the server is transmitting it will send 1-N chunks.
  • end, this is the server signaling there is nothing more to transmit
  • close, this event is raised when the server closes the connection with the server
  • error, this event is raised is something goes wrong

There are more events to listen to but the above should cover the most important cases that might be of interest.

Let's execute the code above. First we start our server and after that we start the client. The client program should now output:

DATA: Chunk received: hello world
END: no more data
CLOSE: Connection closed

As we can see above, our client is able to successfully connect to our server. Then the event data is raised as data is being transmitted. As the total data sent is less than 16 bytes the next event to be raised is end as there is nothing more in the stream to be transmitted. Lastly the event close is raised which is the server closing the connection to the client.

Worth emphasizing is that HTTP is a stateless protocol. As soon as all the data has been sent, the connection is closed. Next time a connection happens there is no memory of any previous connection having taken place.
